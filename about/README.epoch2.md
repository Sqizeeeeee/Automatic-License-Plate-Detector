
# Доработки

### 1. Производительность и Асинхронность
* **Было:** Синхронные вызовы нейросетей YOLO, RapidOCR и функций OpenCV выполнялись в основном потоке бота, полностью блокируя его. Бот обрабатывал запросы в режиме строгой живой очереди; пока один пользователь ждал результат, для всех остальных бот «зависал».
* **Стало:** Внедрена полная асинхронность. Все вычислительные процессы вынесены в фоновые потоки операционной системы через `asyncio.to_thread`. Главный поток бота всегда свободен для мгновенного приема новых сообщений, а обработка картинок от разных пользователей происходит параллельно на уровне ОС.

### 2. Фильтрация текста и Компьютерное зрение
* **Было:** Универсальный движок OCR считывал абсолютно весь текст в кадре. Буквы номера слепо склеивались с мусором, рекламой автосалонов и названиями штатов на рамке (например, `PENNSYLVANIAEXCUSM3`). Применение жестких регулярных выражений ломало распознавание под углами.
* **Стало:** Реализован алгоритм мягкой геометрической и структурной фильтрации. В класс `LicensePlateReader` добавлен статический метод `_get_box_area` для расчета площади текстовых блоков. Мусор отсекается по двум критериям: длина очищенной строки (строго от 3 до 10 символов) и площадь блока (не менее 20% от площади самого крупного текста в кадре). Это убирает рекламу, сохраняя живучесть OCR под любым углом.

### 3. Структура проекта и Чистота кода
* **Было:** Проект состоял из набора разрозненных скриптов. Для связи модулей использовался костыль динамического изменения путей среды через `sys.path.append(...)`. Настройки линтера Ruff хранились в отдельном файле. Зависимости фиксировались в устаревшем `requirements.txt`. Код содержал мелкие нарушения PEP 8 (однострочные `if`).
* **Стало:** Проект переведен на строгую модульную архитектуру пакетов. Весь исходный код изолирован внутри директории `app/`, костыли путей удалены. Управление зависимостями и конфигурация Ruff объединены в один современный декларативный файл `pyproject.toml` (стандарты PEP 518 и PEP 621). Весь код отформатирован линтером Ruff.

### 4. Стабильность системы и UX
* **Было:** Временные файлы картинок сохранялись прямо в корень проекта, загрязняя рабочую директорию. При возникновении любой ошибки внутри OpenCV или OCR бот намертво падал, оставляя мусорные изображения на диске. Во время обработки бот никак не сигнализировал пользователю о процессе.
* **Стало:** Временные файлы изолированы в выделенной папке `app/temp/`. Вся логика обработки обернута в блок `try-except-finally`. При сбоях бот мягко сообщает об ошибке пользователю и продолжает работу, а блок `finally` гарантирует 100% удаление файлов с диска в любой ситуации. Для улучшения UX интегрирован контекстный менеджер `ChatActionSender.upload_photo`, запускающий анимацию «Бот отправляет фото...» во время расчетов.
